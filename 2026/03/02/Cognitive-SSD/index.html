《Cognitive SSD: A Deep Learning Engine for In-Storage Data Retrieval》
总结

为解决数据分析和检索中I/O和数据迁移导致的响应延迟和能耗提升，本文提出了一种基于深度学习的非结构化数据检索引擎——Cognitive SSD，该引擎中的DLG-x闪存访问加速器被放置在闪存旁以实现近数据的深度学习和图搜索。

概述

本文提出了一种整体数据检索机制DLG，将深度学习和图搜索算法相结合，其中深度学习能够提取非结构化数据的语义特征，图搜索能够提高数据库搜索效率。为更好支持在SSD中嵌入DLG，本文设计了一种支持深度哈希和图搜索同时进行的专用硬件加速器DLG-x，从而在不适用功耗不可持续的CPU或GPU解决方案的情况下，构建Cognitive SSD。

相关技术

软硬件协同

在软硬件协同设计方面，SSD的研究有两个方向：开放通道SSD（open-channel SSD）和近数据处理（NDP）。开放通道SSD允许系统软件直接访问闪存，而近数据处理（NDP）则将计算从系统主处理器转移到内存或存储设备中。

板载DRAM

板载DRAM指的是集成在SSD控制器电路板上的动态随机存取存储器（DRAM），也常被称为on-board DRAM或SSD DRAM cache。在SSD中，板载DRAM的主要作用不是做通用计算，而是用于存储和缓存与闪存管理相关的元数据。闪存控制器

闪存转换层

在固态硬盘（SSD）中，FTL（Flash Translation Layer，闪存转换层）是一项核心技术。FTL位于固态硬盘的控制器中，负责管理主机和闪存之间的数据交互，对SSD的性能（Performance）和可靠性（Reliability）起着至关重要的作用。

FTL涉及多个关键功能模块，包括：映射表、坏块管理、垃圾回收、磨损均衡、错误管理等。

DCNN

如图所示，深度卷积神经网络DCNN，其包含四类关键网络层：（1）卷积层：通过在组织成3D张量的输入数据上移动和卷积多维滤波器来提取视觉特征；（2）激活层：对信号进行的非线性转换；（3）池化层：对输入通道进行下采样以实现尺度和其他类型的不变性；（4）全连接层（FC）：在特征和学习到的权重之间执行线性操作，以预测输入数据的分类或其他高层特征。

直接使用FC层的高维输出向量进行数据检索在存储开销和计算复杂度方面代价较高，为此本文采用深度哈希来实现有效且紧凑的数据表示。如上图所示，在AlexNet最后一层后增加一个哈希层，将AlexNet学习到的数据特征映射到哈希空间，生成的哈希码可直接用于索引相关数据结构，从而避免复杂的数据预处理阶段。

Cognitive SSD System

Cognitive SSD系统如图所示，由两个主要部件组成：在轻量级服务器中运行并管理用户请求的DLG库和通过PCIe接口插入主机服务器的Cognitive SSD，Cognitive SSD用以支持DLG框架的主要组件。

DLG 库是基于NVMe协议的I/O命令集中的供应商特定命令 (Vendor Specific Commands) 构建，包括配置库和用户库两部分，如图所示。

配置库使管理员能够根据应用需求，快速在Cognitive SSD上选择和部署不同的深度学习模型。在具有特征提取功能的深度哈希模型部署到Cognitive SSD之后，到达主机服务器的数据处理请求可以通过调用用户库提供的API，向其发送并建立一个查询会话。然后，Cognitive SSD嵌入式处理器上的运行时系统（runtime system）会接收并解析该请求，以激活与用户创建的会话相关联的相应DLG-x模块。

DLG库

配置库

配置库提供了一个与主流深度学习框架（如 Caffe）兼容的DLG-x编译器，以允许管理员离线训练新的深度学习模型并生成相应的DLG-x指令。然后，管理员可以通过更新DLG-x指令来更新在Cognitive SSD上运行的学习模型。更新后的指令被发送到NAND闪存中分配的指令区，并在发出模型变更命令（DLG_config）之前一直保存在该区域中。

同时，在深度学习模型和图的参数被写入NAND闪存之前，DLG-x编译器还会根据神经网络模型和图的结构，对 DLG 算法的数据布局进行重组，以充分利用内部闪存带宽。权重和图结构信息的物理地址被记录在DLG-x指令中。通过这种方式，DLG-x在运行时直接获取所需数据的物理地址，而不是采用会产生额外开销的地址转换或查找操作。

用户库

Data Plane

数据平面为用户提供了 SSD_read 和 SSD_write 两个 API，用于控制主机服务器与Cognitive SSD之间的数据传输。这两个命令直接作用于物理地址，绕过闪存转换层（FTL）。用户可以调用这些API，根据数据地址和数据大小等参数，将来自用户的数据注入到Cognitive SSD的数据缓存区域或NAND闪存中。之后，用户可以在其他 API 中使用这些地址来指定操作数。

Task Plane

为提升DLG-x加速器的可扩展性，在任务平面中DLG-x的功能被抽象为三个API：DLG_hashing、DLG_index 和DLG_analysis。它们分别基于NVMe I/O协议中的C0h、C1h和C2h命令实现。所有这些API均通过NVMe协议的 DWord携带两个基本参数：指示Cognitive SSD中数据位置的数据地址和以字节为单位的数据大小。

  • DLG_hashing API：用于提取输入数据的压缩特征，并将其映射到哈希或语义空间。该命令包含一个扩展参数：哈希码长度，用于决定所携带信息的容量。例如，与包含500种对象类型的数据库相比，包含1000种对象的数据库需要更长的哈希码以避免信息丢失。
  • DLG_index API：抽象自DLG-x的图搜索功能。它包含一个扩展参数T，代表用户根据应用场景配置的搜索结果数量。
  • DLG_analysis API：允许用户利用深度神经网络的数据分析和处理能力来分析输入数据，并预留了一个字段用于用户自定义功能。

上述任务API是Cognitive SSD提供的关键近数据处理核心的抽象，它们可以被独立或组合调用，以开发不同的 SSD 内数据处理功能。例如，用户可以组合DLG_hashing和DLG_index API来完成在大型数据库上的数据检索，其中DLG_hashing将查询数据的特征映射为哈希码，而DLG_index使用该哈希码搜索前T个最相似的实例。

Cognitive SSD Runtime

Runtime system被部署在Cognitive SSD内部嵌入式处理器上，负责管理通过PCIe接口接收的扩展I/O命令，同时将与API相关的命令转换为DLG-x加速器可执行的机器指令并处理NAND闪存的基本操作。

Cognitive SSD Runtime包括请求调度器和基础固件。请求调度器由三个模块组成：DLG任务调度器、I/O调度器和DLG配置器。基础固件包含用于逻辑块映射、垃圾回收和坏块管理等功能的闪存转换层，并与NAND闪存控制器通信以处理通用I/O请求。

  • DLG配置器：从主机接收DLG_config命令，用于更新编译器生成的指令以及指定深度学习模型的参数。
  • DLG任务调度器：响应任务平面中支持的用户请求，并在Cognitive SSD中启动相应的任务会话。
  • I/O调度器 将I/O请求分派给基础固件或DLG-x加速器。

DLG-x 加速器在激活后会占用相当一部分闪存带宽，可能会降低普通I/O请求的性能。为缓解这一问题，DLG任务调度器通过doorbell机制接收来自主机的NVMe命令，并周期性地主动轮询DLG-x的完成状态，以决定是否可以分发下一个请求。相比于让任务调度器或I/O调度器等待请求完成的方式，该方法能有效缓解普通I/O带宽降低。

硬件架构

Cognitive SSD的硬件架构如图所示：

Cognitive SSD由一个运行Cognitive SSD运行时系统的嵌入式处理器、一个DLG-x加速器和连接闪存芯片的NAND闪存控制器组成。每个NAND闪存控制器连接一个NAND闪存模块的通道，并使用一个ECC引擎进行纠错。SSD中有限的DRAM主要用于闪存管理的元数据缓存，几乎没有空闲空间供深度学习应用使用。当每个通道的设备以锁步方式并行访问时，内部带宽超过I/O接口的带宽。因此，NAND闪存控制器被暴露给DLG-x加速器，使得DLG-x能够直接从NAND闪存中读取和写入相关的工作数据，从而绕过内部DRAM。

Cognitive SSD中的数据检索过程

当用户执行非结构化数据检索任务时，Cognitive SSD的整体处理流程：

  1. 首先，假设Hash-AlexNet模型的硬件指令和参数已经通过利用DLG-x编译器和图中所示的DLG_config命令生成并写入到相应的区域。Hash-AlexNet是开发人员为输入数据特征提取而指定的神经网络。
  2. 然后，当主机DLG库捕获到一个检索请求时，它通过SSD_write API将用户输入数据从指定的主机内存空间打包并写入Cognitive SSD。同时，携带输入数据地址的DLG_hashing命令被发送到Cognitive SSD以生成哈希码。
  3. 接收到该命令后，Cognitive SSD Runtime的请求调度器会解析它，并通知DLG-x加速器启动一个哈希特征提取会话。然后，DLG-x自动从命令指定的数据地址获取输入查询数据，接着从NAND闪存加载深度学习参数。
  4. 与此同时，另一个命令DLG_index被发送并由任务调度器排队。在哈希码生成后，DLG_index被分派以调用DLG-x中的图搜索功能，并使用哈希结果在数据图中搜索相关的数据条目。在这种情况下，DLG-x会持续从NAND闪存中获取图数据，并在任务完成后将最终的检索结果发送到主机内存。

DLG-x Accelerator

DLG-x架构

DLG-x加速器被设计为直接从NAND闪存中获取大部分工作集数据，其结构如图所示：

DLG-x加速器包含两个激活缓冲区（InOut Buffer）和双排的权重缓冲区。各神经网络层的中间结果暂存于激活缓冲区，权重缓冲区则作为神经处理引擎（NPE）与闪存间的桥梁缓冲区，将大量神经参数流式传输至NPE。

NPE由一组处理单元（PEs）构成，能够执行定点卷积、池化以及激活函数等运算。图搜索引擎（GSE）与NPE协同工作，负责利用NPE生成的哈希码执行图搜索操作。NPE与GSE均由控制单元管理，该单元从存储器中获取指令。

Data Layout In Flash Devices

本文提出了一种闪存感知的数据布局方案，利用NAND闪存提供的页缓存顺序读取命令以充分发挥闪存带宽。该命令允许在前一页数据从缓存寄存器读取到DLG-x缓冲区或SSD缓存区域的同时，将同一块中的下一页连续加载到数据寄存器中。基于支持页缓存指令的NAND闪存架构，本文将卷积核进行分割，并将其存入闪存设备中，以实现并行读取。

如图所示，假设有N_k个卷积核，其核大小为S_k，且DLG-x加速器使用M个NAND闪存通道，则每个卷积核被分割为S_k/M个子块，所有这些子块以交错方式存储到各个闪存通道中。对于大小超过单个页面的卷积核，它们被放置在NAND闪存的连续地址空间内，因为缓存指令会自动读出下一页，无需任何额外的地址或操作。

数据流

以Hash-AlexNet为例，当一个请求到达DLG-x加速器时，输入数据以及第一卷积层的第一个卷积核，被并行传输至输入/输出缓冲区-0和权重缓冲区-0。随后，DLG-x加速器开始计算输出特征图，并将其存储到InOut缓冲区-1中。当第一个卷积核处理完毕时，第二个卷积核从NAND闪存传输至权重缓冲区-1，之后依次加载第三个和第四个卷积核。一旦哈希码生成完成，若DLG任务调度器解析并分发了后续的DLG_index命令，该哈希码将被送入NPE的图搜索寄存器中，用于定位与查询数据相似的数据结构。

深度学习和图搜索融合

DLG-x加速器将深度学习与图搜索机制融合，并复用计算与存储资源以提升整体效率。在查询数据的哈希码生成之后，DLG-x利用该哈希码对相应的数据图进行初始索引，并在图中搜索最接近的数据条目。

DLG-x采用NSG算法进行图搜索，该算法包含离线和在线阶段。在离线阶段，NSG为向量数据构建有向最近邻图K_nbors。图中每个顶点表示一个数据条目，顶点中保存其唯一标识符ID和哈希码。唯一ID对应一个文件，哈希码为该文件的特征向量，可通过预先调用DLG_hashing API获得。ID位宽和哈希码位宽可在API中进行配置。邻居数量K_nbors同样可以进行配置，以权衡检索精度和检索速度。

如图所示，权重缓冲区和InOut缓冲区分别被复用于存储顶点的邻居信息以及图搜索引擎的搜索结果。相似度计算采用汉明距离，InOut缓冲区按照汉明距离的取值范围划分为多个区块。

在图搜索过程中，系统引入了顶点检测单元（VDU）用于检查所选顶点是否已经被访问过。若发现该顶点此前已被遍历，则直接丢弃；否则，将其送入NPE单元计算其与查询顶点之间的汉明距离。在NPE提供距离信息后，缓冲区地址生成器（BAG）模块为该顶点在InOut缓冲区中分配存储空间，并将其放入对应的缓冲区区域中。尚未评估的顶点会从InOut缓冲区中取出，其邻居信息则由控制单元从权重缓冲区中加载。当InOut缓冲区中的顶点数量达到用户配置的阈值时，控制单元最终返回距离最近的前 T 个顶点，其中T同样由用户通过DLG_index API进行配置。

Data Layout for fast In-SSD NSG search

NAND闪存的读操作以页为粒度（16 KB）执行，因此，每当DLG-x访问某一顶点的邻域数据（约 250 字节）时，都必须从闪存中读取一个完整页面。为保持良好的空间局部性，将顶点V0及其所有邻居（V1,V2,…,V25）连续对齐并存储在同一页的起始位置，如图所示：

采用这种冗余的数据布局方式，以牺牲额外存储空间来换取更高的数据访问性能。

数据流

基于DLG-x的数据检索整体流程：

  1. 首先，当一个查询到达时，DLG-x将输入数据和深度学习模型参数分别从NAND闪存加载到InOut缓冲区和权重缓冲区。
  2. 随后，NPE单元为输入数据生成哈希码，并将其写入NPE的图搜索寄存器中。接着，DLG-x将K_nbors最近邻图从NAND闪存阵列传输至权重缓冲区。
  3. 在第一阶段，计算初始顶点并将其送入InOut缓冲区中相应的区域。在第二阶段，DLG-x控制单元按照汉明距离从小到大的顺序，从InOut缓冲区中读取首个尚未评估的顶点。
  4. 图搜索引擎从NAND闪存中获取该顶点的邻居，并将这些邻居送入NPE计算其与查询顶点之间的汉明距离。
  5. 缓冲区地址生成器根据计算得到的汉明距离，为这些邻居顶点生成InOut缓冲区的写入地址，并将其写入缓冲区。同时，图搜索引擎中的计数器通过监测InOut缓冲区中已存储顶点的总数，判断是否需要发出终止信号。
  6. 一旦生成终止信号，Cognitive SSD Runtime系统便从InOut缓冲区中读取顶点信息，并通过PCIe接口将存储在NAND闪存中的、由ID指向的检索结果传输回主机服务器。
